what it is

Enterprise containment for persistent digital workers.

Instead of exposing your company networks to raw API keys and unchecked scripts, this substrate hosts persistent agent employees inside an isolated, highly-auditable virtual environment. Running inside your Slack, Discord, or ClickUp channels, the agent executes complex tasks securely on the VM.

It is deliberately agent-agnostic. Any digital worker enters as a manifest and a risk tier, then lands in a secure, default-deny hosting container with a continuous, unified context—allowing it to safely manage files, edit Excel sheets, and join meeting calls compliant with enterprise policies.

cooperative business use cases

A highly governed team member, not an isolated chatbot tab.

01

Repeatable workflows

Instead of solitary chatbot tabs, add the agent employee directly to shared team channels (Slack, Discord, ClickUp). Employees collaborate normally with each other while delegating complex, repeatable workflows directly to the agent in chat.

02

Persistent secure context

The agent keeps continuous context of the team's ongoing projects because its shared files live persistently on the secure VM disk. It can ingest data, read on-disk files, and deliver polished Excel spreadsheets or PowerPoints conforming to strict branding guidelines directly back to the channel.

03

Compliant meeting deliverables

The agent employee securely joins meeting calls (Google Meet/Zoom) to pull transcripts, processing them locally on the VM substrate to generate meeting notes, action items, or direct work assets compliant with enterprise data protection guidelines.

04

Enterprise-grade boundaries

By wrapping the agent in a secure VM substrate, businesses can safely delegate real-world execution. Default-deny network policies, hard timeouts, and credential-by-reference gates ensure tasks run safely without exfiltration risk.

three layers + governance

Every layer assumes the one above it can fail.

01

Promotion control plane

Immutable exact-commit releases, dry-run-by-default mutation, drift detection, state as truth, and recorded rollback targets.

  • exact commit promotion
  • dry-run before apply
  • drift-aware status
02

Isolation substrate

Nested-virt golden VM with Tier-1 cosign-signed, digest-pinned services and Tier-2 ephemeral Kata microVM jobs.

  • reconcile → align
  • default-deny egress
  • verified teardown
03

agent-gateway@<profile>

Per-profile runtime layout with one declared runtime source, OS-level profile isolation, and independently restartable hardened systemd units.

  • one service per profile
  • Linux user isolation
  • restart blast-radius control
G

Governance overlay

Risk tiers L0–L5, SLO/canary, fail-closed MCP tool allowlists, secrets-by-reference, signed supply chain, audit trails, and proven rollback.

  • risk-tier gates
  • canary before promotion
  • audit evidence

reference workloads

Two digital worker shapes, one hosting contract.

Archetype A

Immutable-release collaborator

Python-based persistent worker promoted by exact git commit, continuously monitoring group chats (Slack, Discord) to handle complex, repeatable task flows.

Archetype B

Task-on-demand agent pool

Package-install worker pool that spins up in ephemeral microVM sandboxes to securely digest transcripts or render large file assets without lingering side effects.
